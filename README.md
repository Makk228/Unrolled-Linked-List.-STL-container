# Unrolled Linked List — STL-совместимый контейнер

Проект представляет собой реализацию STL-совместимого контейнера **Unrolled Linked List** (разреженный связный список блоков). Контейнер сочетает свойства списка и массива: элементы хранятся в узлах-блоках фиксированной ёмкости, что уменьшает накладные расходы на память и повышает локальность данных.

Ссылка на описание структуры: [Unrolled linked list](https://en.wikipedia.org/wiki/Unrolled_linked_list).

## О проекте

**Unrolled Linked List — STL-совместимый контейнер** реализован как шаблон, параметризуемый:

- типом хранимых объектов;
- максимальным количеством элементов в узле;
- аллокатором.

Проект описывает контейнер, который частично соответствует named-requirements стандартной библиотеки C++ и демонстрирует корректные гарантии сложности и исключений для ключевых операций.

## Совместимость с STL

Контейнер ориентирован на следующие named-requirements:

- [Container](https://en.cppreference.com/w/cpp/named_req/Container)
- [SequenceContainer](https://en.cppreference.com/w/cpp/named_req/SequenceContainer) (без `emplace`, `assign_range`, `emplace_front`, `emplace_back`, `prepend_range`, `operator[]`)
- [ReversibleContainer](https://en.cppreference.com/w/cpp/named_req/ReversibleContainer)
- [AllocatorAwareContainer](https://en.cppreference.com/w/cpp/named_req/AllocatorAwareContainer)
- [BidirectionalIterator](https://en.cppreference.com/w/cpp/named_req/BidirectionalIterator)

## Интерфейс и сложность

Ниже перечислены операции контейнера с их асимптотикой и гарантиями исключений:

| Метод       | Алгоритмическая сложность          | Гарантии исключений |
|------------|------------------------------------|---------------------|
| `insert`   | `O(1)` для 1 элемента, `O(M)` для M | strong              |
| `erase`    | `O(1)` для 1 элемента, `O(M)` для M | noexcept            |
| `clear`    | `O(N)`                              | noexcept            |
| `push_back`| `O(1)`                              | strong              |
| `pop_back` | `O(1)`                              | noexcept            |
| `push_front`| `O(1)`                             | strong              |
| `pop_front`| `O(1)`                              | noexcept            |

> Примечание: `M` — количество элементов, затронутых операцией внутри узла.

## Концепция реализации

Ниже — ключевые идеи, на которых строится реализация контейнера:

- **Узлы-блоки**. Каждый узел хранит массив элементов фиксированной ёмкости и число фактически размещённых объектов. Это позволяет выполнять вставки/удаления внутри узла за `O(1)` для одиночного элемента и за `O(M)` при сдвиге диапазона внутри блока.
- **Связи между узлами**. Узлы объединяются в двусвязный список (prev/next), обеспечивая эффективные операции на концах и двунаправленное движение итератора.
- **Инварианты заполненности**. После вставок и удалений контейнер поддерживает корректное распределение элементов по узлам (например, перераспределение или слияние при переполнении/опустошении блока), сохраняя заданную сложность операций.
- **Итераторы**. Итератор содержит указатель на текущий узел и индекс элемента внутри блока. Переход на следующий/предыдущий элемент — это либо смещение индекса, либо переход на соседний узел при достижении границы блока.
- **Учет аллокатора**. Память под узлы и элементы выделяется через пользовательский аллокатор; уничтожение и освобождение происходит через `allocator_traits` для корректной поддержки пользовательских типов.

## Итераторы и аллокаторы

Контейнер поддерживает:

- двунаправленные итераторы;
- обратные итераторы;
- корректную работу с пользовательским аллокатором (Allocator-aware).
